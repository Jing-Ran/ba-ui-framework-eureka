Herald Component

Inspired by the heralds of yore who conveyed a monarch's proclamation, the main attraction of The Herald is the unfurling border and the dropped message with a banner. The component features a 400x200 image (or any 2:1 sized image if scaling up is desired) and a place for textual content. Ideally, The Herald is useful for small listings, although any message size will fit inside The Herald component because the height is determined and set dynamically.

To use:

I. Copy the markup below, then place announcements inside the c-herald__message tag. Each unique announcement must be placed within <p> tags. Next, add the number of announcements inside c-chip__media--o. Inputting the number of announcements is recommended in case users have disabled JavaScript. Note: The number of announcements in c-chip__media--o are based on the number of <p> tags. This number is updated dynamically in the event the number of announcements was not inputted.

II. The ideal size image is 400x200 pixels for the normal-size component. For a larger Herald, scale up the component by setting a larger font-size on c-herald, then use a 2:1 sized image that is proportionate to the larger component. For example, if a font-size of 2em is added as a property to c-herald, the image should be doubled to 800x400 pixels.

Accessibility: In the event a user disables JavaScript, The Herald and all content inside it will be shown. There will be no animation and dynamic counting of a message's <p> tag, however. This is due to the nature of how the component resizes based on the message element's offsetHeight value.

Incompatibilities: border-top of the content div does not display in Safari.
